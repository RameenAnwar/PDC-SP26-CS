## Concept Summary

### Classes & Objects

Defines structure of program using class and objects.
Class variables are shared, instance variables are unique.

### Control Flow

Used for decision making and looping:

- if-else
- for loop
- while loop

### Data Structures

- List → mutable
- Tuple → immutable
- Dictionary → key-value

### File Handling

Reading and writing data using files for permanent storage.

### Functions & Modules

Reusable code blocks and importing functions across files.

### Execution Models

- Serial → one by one (slow)
- Multithreading → shared memory, fast for I/O
- Multiprocessing → separate memory, best for CPU

## Applications

- OOP systems
- Data storage
- API handling
- Heavy computations

## Advantages

- Reusability
- Better performance
- Structured code

## Disadvantages

- Complexity increases
- Threading issues (race condition)
- High memory (multiprocessing)

## Conclusion

Chapter 1 covers Python fundamentals and shows how parallel execution improves performance over serial processing.
